# CrabWork 设计协作

面向设计团队的 CrabCode 插件:设计评审、设计系统管理、UX 文案、无障碍审查、用户研究综合与开发交付。可独立使用(描述设计、粘贴截图、上传素材),连接 Figma 及其它工具后能力更强——从早期探索到像素级规格交付一站搞定。

> 基于 [anthropics/knowledge-work-plugins](https://github.com/anthropics/knowledge-work-plugins)(Apache-2.0)二次开发,已去品牌化并适配 CrabCode 生态。

## 安装

在 CrabCode 插件市场中搜索「CrabWork 设计协作」并安装,或通过 marketplace 添加 `crabwork-design`。

## 技能(按需自动触发)

CrabCode 会根据你的输入自动匹配以下技能:

| 技能 | 说明 |
|---|---|
| `design-critique` | 结构化设计评审:可用性、视觉层级、一致性与设计原则符合度 |
| `design-system` | 审计、记录或扩展设计系统:组件、设计令牌、模式文档 |
| `design-handoff` | 生成开发交付规格:测量值、令牌、组件状态、交互与边界情况 |
| `ux-copy` | 撰写或评审 UX 文案:微文案、错误提示、空状态、引导流程 |
| `accessibility-review` | 无障碍审查:WCAG 2.1 AA 合规、色彩对比、屏幕阅读器与键盘导航 |
| `user-research` | 规划、执行与综合用户研究:访谈、问卷、可用性测试 |
| `research-synthesis` | 综合用户研究素材:从访谈、问卷、可用性测试中提炼可行动的洞察 |

## 示例工作流

### 获取设计反馈

分享 Figma 链接、截图或描述你的设计,获得关于可用性、视觉层级、一致性与无障碍的结构化反馈。

### 审计设计系统

针对组件库进行一致性、完整性与命名规范审查,产出带具体改进建议的报告。

### 撰写 UX 文案

例如「为支付流程撰写错误提示」,获得贴合语境的文案、语气指引、备选方案与本地化注意事项。

### 开发交付

分享 Figma 链接,获得完整规格:测量值、设计令牌、组件状态、交互说明与边界情况。

### 无障碍检查

分享设计或 URL,获得 WCAG 2.1 AA 合规报告:具体问题、严重程度与整改步骤。

### 综合研究

上传访谈记录、问卷结果或可用性测试笔记,获得主题归纳、洞察与排序后的建议。

## 独立使用 + 连接增强

每个技能无需任何集成即可使用;连接 MCP 工具后体验更佳:

| 能力 | 独立使用 | 连接增强 |
|---|---|---|
| 设计评审 | 描述或截图 | Figma(直接拉取设计) |
| 设计系统 | 描述你的系统 | Figma(审计组件库) |
| 交付规格 | 描述或截图 | Figma(精确测量值与令牌) |
| UX 文案 | 描述语境 | 知识库(品牌语调规范) |
| 无障碍 | 描述或截图 | Figma、产品分析(真实使用数据) |
| 研究综合 | 粘贴访谈记录 | 用户反馈工具(拉取原始数据) |

## MCP 连接器

> 如遇占位符或需确认已连接的工具,请参阅 [CONNECTORS.md](CONNECTORS.md)。

| 类别 | 示例 | 启用能力 |
|---|---|---|
| 设计工具 | Figma | 拉取设计、检查组件、读取设计令牌 |
| 用户反馈 | Intercom、Productboard | 原始反馈、功能请求、NPS 数据 |
| 项目管理 | Linear、Asana、Jira | 关联设计与工单、跟踪落地进度 |
| 知识库 | Notion | 品牌规范、设计原则、研究资料库 |
| 产品分析 | Amplitude、Mixpanel | 用于研究综合与设计决策的使用数据 |

详见 [CONNECTORS.md](CONNECTORS.md) 获取完整的受支持集成列表。

## 个性化设置

在 `crabwork-design/.crabcode/settings.local.json` 创建本地设置文件:

```json
{
  "name": "你的名字",
  "title": "产品设计师",
  "team": "你的团队",
  "company": "你的公司",
  "designTool": "Figma",
  "brandVoice": "简洁、友好、专业"
}
```

未配置时,插件会在需要时主动询问相关信息。
