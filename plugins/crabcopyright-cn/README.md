# 软著申请管家 · crabcopyright-cn

面向**中国版权保护中心**软件著作权登记的知识产权材料管理工作流伞插件。把原来一整块的
"软著申请"脚本拆成伞状:一个总编排"管家"负责统筹调度,各专职技能负责一道工序。属于
**知识产权材料管理**范畴,与法律实务工作流(crablaw-cn)分家。

## 结构（按职能分 2 组）

### 组一 · 申请编排（apply-core）
| 技能 | 职责 |
|------|------|
| `apply-manager` | **软著申请管家**——总入口/路由,规划全流程、判断阶段、分派子技能、守全局一致性红线 |
| `application-planning` | **第 0 步**:按第一性原理分析本地仓库,规划拆成几个软著、每块归属哪些源码、说明书截哪些前端页面(含开发环境访问地址) |
| `materials-checklist` | 按申请人类型与开发方式核算所需材料,标注已有/缺失/待确认 |
| `consistency-check` | 逐字校验名称/版本号/日期在各材料中一致(软著最高频驳回点) |
| `package-build` | 按命名规范归档 + 生成材料自查对照表 |
| `filing-guide` | 平台在线填报逐字段卡片 + 2026 新版申请表提醒 |

### 组二 · 鉴别材料制作（identification-material）
| 技能 | 职责 |
|------|------|
| `source-code-material` | 扫描源码,生成 60 页规范源程序鉴别材料 PDF |
| `manual-material` | 整理 Word 说明书为规范文档鉴别材料 PDF |

### 子代理（agents/,只读,由主会话 Task 派发）
| 子代理 | 职责 |
|--------|------|
| `sc-material-collector` | 收集:扫仓库挑源码文件、统计行数,回传候选清单 |
| `manual-evidence-collector` | 收集(可选):核对细化说明书截图取证清单 |
| `filing-reviewer` | 对抗式审查:独立做一致性校验+跨申请查重,调用 scripts/ 脚本,只报告不修改 |

信任隔离:收集/审查子代理一律只读;manifest 与最终材料只由主会话工序落盘
(最终材料仅 package-build 写)。多软著批量由 apply-manager 按申请 fan-out 并行派发。

### 确定性校验脚本（scripts/,python3 纯标准库,零依赖）
行数/页数折算、功能说明字数、注水启发式、跨申请查重、日期逻辑均由脚本机械判定,
`check_all.py` 为总入口,均支持 `--json`;package-build 的自查对照表由脚本结果产出,
不靠模型自述勾选。脚本用法与 manifest 结构见 `apply-core/MANIFEST.md`。

共享事实源:`apply-core/GUIDE.md`(官方规范速查)——所有页数/行数/日期/红线以它为准,
区分【官方】规定与【经验做法】。工序间参数交接读写 `outputs/<申请名>/manifest.json`
(结构见 `apply-core/MANIFEST.md`),不靠口头复述。

## 与 crabcode-office-suite 办公套件的协作

本插件**不自带**文档处理能力,而是引导软著工作流在合适场景调用办公套件:
- 读/改/排版 Word 说明书、插封面目录页眉页码截图 → `crabcode-documents`
- 定稿转 PDF、合并、加页码、按前30+后30页取页 → `crabcode-pdf`
- 说明书内数据表格 → `crabcode-spreadsheets`

## 使用

对管家说"帮我申请软著""准备软著材料""把这个项目登记软件著作权"即可触发 `apply-manager`,
由它规划并分派;也可直接触发某道工序(如"整理软著源代码""软著说明书怎么排版")。

## 重要提示

- 官方普通登记自 **2017-04-01 起免费**;仅委托代理才有代理费。
- **一致性**是最高红线:软件全称、版本号、日期在申请表/源码页眉/说明书三处逐字一致。
- **2026 新版申请表**(经办人本人签字+身份证号、手抄"未使用 AI"诚信承诺、功能说明 500–1300 字)以填报时平台实际字段为准,非《登记办法》条文。
- 绝不伪造/PS/注水;产物不留身份证号等隐私,由用户自行填入平台。
