# 软著登记官方规范速查（单一事实源）

本文件是「软著申请管家」全部技能共享的事实基准。凡涉及页数、行数、日期逻辑、
材料清单、红线的判断，都以此处为准，不要各技能各写一套、彼此打架。

> 术语约定：下文用【官方】标注《计算机软件著作权登记办法》原文或中国版权保护中心官网
> 明列的硬性规定；用【经验做法】标注仅见于代理机构/行业整理的通行做法（可作默认值，
> 但遇到平台实际提示不一致时以平台为准）。这个区分很重要——把经验做法当成铁律去
> 卡用户，会误导；把官方硬规当经验忽略，会被驳回。

---

## 1. 法律框架与"三大件"

【官方】《计算机软件著作权登记办法》第七条：向**中国版权保护中心**提交——
1. **软件著作权登记申请表**（网上填报生成，签章页签字/盖章）；
2. **鉴别材料**＝**源程序** ＋ **文档**（说明书/手册任选一种）；
3. **证明文件**＝身份证明 ＋（特殊情形的）权属证明。

- 官网所需文件：https://www.ccopyright.com.cn/index.php?optionid=1080
- 登记平台入口：https://register.ccopyright.com.cn/registration.html

## 2. 身份证明与特殊情形补充材料

### 2.1 身份证明【官方】
| 申请人类型 | 身份证明 |
|---|---|
| 自然人（个人） | 身份证正反面；外国自然人：护照 ＋ 中文翻译件 |
| 企业 | 加载统一社会信用代码的营业执照副本 |
| 事业单位/机关/社团 | 对应法人证书 |

### 2.2 特殊情形补充材料【官方，《登记办法》第八条】
| 情形 | 需补充 |
|---|---|
| 合作开发 | 合作开发合同（证明归属约定） |
| 委托开发 | 委托开发合同 |
| 下达任务开发 | 项目任务书/合同 |
| 继受取得（受让/转让） | 权利转让合同 ＋ 原始权属证明（原证书） |
| 继承取得 | 继承权证明 |
| 企业合并/更名 | 企业变更、合并文件 |
| **在原有软件上二次开发/修改** | **经原著作权人许可的，须提交原著作权人的许可证明**；实质性改动按新软件登记 |

## 3. 源程序鉴别材料格式

- 【官方，第九条】源程序＋文档各取**前 30 页 ＋ 后 30 页 ＝ 60 页**；整个程序不足 60 页的，**提交全部**（不得用空行/注释凑）。
- 【官方】**每页不少于 50 行**（末页、含图页可例外）。
- 【经验做法】第 1 页为程序/功能模块的**开头**，第 60 页为**结尾**，体现完整性；末页呈现自然的程序结尾（函数/类闭合、`}`、`return`、`END`），不要在语句中间截断。
- 【官方一致性】页眉标注的**软件名称＋版本号**必须与申请表、文档**逐字一致**。
- 【经验做法】页码：**右上角**连续阿拉伯数字，第 1→60 页连续。字体等宽、常用小四号、单倍行距以保证每页 ≥50 行。
- 【红线】禁止**代码注水**：大段空行、纯注释、重复模板凑行数/页数，会被认定注水驳回；以有效代码为主。
- 【官方，涉密】核心秘密可走**例外交存**（如前后各 10 页＋中间任选连续 30 页、或目标程序＋源程序组合等），需申请时特别声明，建议咨询登记机构。
- 【平台】线上提交转 **PDF** 上传。

---

## 4. 文档鉴别材料（说明书/手册/设计说明书）格式

- 【官方】提交**任意一种**文档即可，通常三选一：
  - **用户手册/操作说明书** —— 适合有 UI 的应用类软件（登录界面＋主界面＋各功能界面截图＋对应文字）；
  - **设计说明书** —— 适合无 UI 的底层/算法/嵌入式（模块化文字＋流程图/结构图）；
  - 需求文档/程序流程图等。
- 【官方】文档同样**前后各连续 30 页**（不足 60 页交全部）；**每页不少于 30 行**（图片页、末页可例外，截图页不受 30 行限制）。
- 【经验做法】常见章节：**封面 → 目录 → 概述（开发目的/简介）→ 运行环境（软硬件/OS）→ 安装说明 → 功能模块操作说明（配截图）→ 注意事项**。设计说明书：引言 → 总体设计 → 逐模块程序描述（功能/输入输出/算法/接口/限制）。
- 【经验做法】**封面不计页数**；页码从目录或正文首页起、右上角连续。截图之间有文字说明，图文并茂、每功能配图。
- 【红线】文档中的软件名称、版本号、**功能描述必须与源代码、申请表吻合**——文档写了 A 功能，源码里要能找到 A，否则驳回。
- 【经验做法】文档中的网址建议用 `localhost`/`127.0.0.1`，避免暴露真实域名引审查疑问。

## 5. 名称与版本号规范

- 【官方】**软件全称**：简短、明确、有针对性；建议"品牌 ＋ 用途/功能 ＋ 软件/系统/平台"（如"XX 企业进销存管理系统"）。不得含行政区划、夸大用语、侵犯商标；各材料全称逐字一致。
- 【官方】**软件简称**：全称的简化名，不能与全称完全相同；没有可不填。
- 【官方】**版本号**：`V1.0` 或 `1.0` 两种写法均可，但**所有材料写法必须统一**（有无"V"要一致）；首次登记通常 V1.0。

## 6. 申请表关键字段与日期逻辑

| 字段 | 要点【官方填写说明】 |
|---|---|
| 软件全称/简称 | 见 §5，全材料一致 |
| 版本号 | 见 §5，写法统一 |
| **开发完成日期** | 软件**全部固定在有形载体**上的日期；如实填 |
| **首次发表日期** | 首次公之于众（销售/发布/展示）之日；**不得早于开发完成日期**；未发表则选"未发表"、不填日期 |
| 权利取得方式 | 原始取得（自主/合作/委托/职务）或继受取得（受让/继承）；继受须附权属证明 |
| 开发方式 | 独立/合作/委托/下达任务（对应 §2.2 补充材料） |
| 分类号 | 按国标软件分类代码选填（较专业） |
| 著作权人 | 与身份证明一致；单位须与营业执照全称一致 |
| 经办人 | 见 §8 新规 |

**日期红线**：开发完成日期 ≤ 首次发表日期；企业申请时开发完成日期不得早于企业成立日期。

## 7. 费用与周期

- 【官方，重要】中国版权保护中心自 **2017 年 4 月 1 日起停止征收软件著作权登记费**——**普通登记官方免费**（注：坊间"2024 年起停征"是误传，正确停征日为 2017-04-01）。委托代理另付代理费（非官费）。
- 【官方】法定审限：**受理日起 60 日内**审查完成。【经验做法】普通件实务约 **30–60 个工作日**取证；市场有付费加急代办。

## 8. 2026 新版申请表要求（多来源一致，但非《办法》条文——以最新版申请表与平台实际提示为准）

> **可靠性说明**：以下内容据湖北省版权中心转述、媒体报道、多地软件行业协会解读，
> 多来源高度一致、可信度较高，但**中国版权保护中心官网缺乏可逐字援引的稳定公告链接**。
> 因此技能中应把它表述为"2026 新版申请表要求"，**不要冒充《登记办法》条文**；
> 提醒用户以填报时申请表实际字段/平台提示为准。

- 自 **2026-03-15** 起启用新版申请表，建立"版权登记申请诚信制度"。
- 签章页新增**手抄承诺**，大意："本软件确系独立开发，未使用 AI 开发编写代码、撰写文档或生成登记申请材料"，并附失实责任条款。
- **经办人/申请人须本人签名并附身份证号**（禁止代签）。
- 违规者列入**版权登记失信名单并关联个人征信**。
- 软件主要功能说明字数要求从"≤200 字"提高到约 **500–1300 字**。
- 若确有 AI 辅助编码，建议保留真实开发记录、并以最新版申请表口径如实处理。

## 9. 常见驳回红线（核对清单）

1. **材料不一致**（最高频）：源码/文档/申请表的名称或版本号不统一。
2. **代码注水**：空行、纯注释、重复模板凑行数。
3. **说明书模板化**：套模板、截图与功能对不上、描述空泛。
4. **名称不规范**：含行政区划、夸大、侵犯商标、全称=简称。
5. **日期逻辑错**：首次发表早于开发完成；日期随意填。
6. **页数/行数不足**：不足 60 页却未交全部；每页行数不达标。

**绝不**伪造、PS、虚构任何材料内容；**不**在产物中保留身份证号等隐私，由用户自行填入。
