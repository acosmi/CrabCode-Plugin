# 软著申请管家（总编排）

统筹一次完整的软件著作权登记申请,产出一套可提交至**中国版权保护中心**
(register.ccopyright.com.cn) 的电子材料包。本技能是伞状插件的入口与调度中枢:
自己不埋头做具体材料,而是规划全局、判断用户卡在哪一步、把活分派给专职子技能。

**开工前先读** `${CRABCODE_PLUGIN_ROOT}/apply-core/GUIDE.md`(官方规范速查)——所有页数、
行数、日期、红线判断都以它为准,不要凭记忆另编一套。

## 输入（缺则用 AskUserQuestion 索取）

| 参数 | 说明 | 示例 |
|------|------|------|
| `$SOFTWARE_NAME` | 软件全称 | `晶石进销存管理系统` |
| `$VERSION` | 版本号(`V1.0` 或 `1.0`,全材料统一) | `V1.0` |
| `$SOURCE_CODE_DIR` | 源代码根目录 | `/path/to/project` |
| `$MANUAL_PATH` | 说明书 doc/docx 路径(可后补) | `/path/to/manual.docx` |
| `$APPLICANT_TYPE` | 申请人类型 | `个人`/`企业`/`事业单位` |

## 最终产物

```
软著申请-${SOFTWARE_NAME}${VERSION}/
├── 01-软件著作权登记申请表.pdf     （用户在平台生成后放入）
├── 02-源代码鉴别材料.pdf           （前30+后30页,页眉含名称+版本号）
├── 03-说明书鉴别材料.pdf           （封面/目录/概述/功能截图/运行环境）
├── 04-身份证明文件.pdf             （用户自备）
├── 05-其他材料/                    （合作/委托/二次开发等补充）
└── 材料自查对照表.md
```

## 编排流程（判断阶段 → 点名下一道工序）

先弄清用户已经到哪一步,再从对应环节切入;不必每次都从头走。本技能不直接代跑
子技能,而是**按顺序点名下一道工序并交接参数**——各子技能都有自己的触发词,会在
对应场景被唤起。标准顺序:

0. **申请板块规划** → 转 **application-planning** 技能
   若源码是 monorepo/大项目,或用户想登记**多个**软著,**先做这步**:分析仓库结构,
   按第一性原理规划拆成几个申请、每个申请归属哪些源码目录、说明书截哪些前端页面
   (含开发环境访问地址)。规划确定后,**对每个申请循环走下面 1–6**——每个申请是
   独立的(全称+版本),彼此不共用页眉名称、不重复计入公共代码。单一软件可跳过此步。

1. **材料清单核查** → 转 **materials-checklist** 技能
   按 `$APPLICANT_TYPE` 和开发方式,列出该情形所需全部材料与缺口。**先做这步**,
   因为它决定后面要不要准备许可证明、合同等补充件。

2. **源代码鉴别材料** → 转 **source-code-material** 技能
   扫描 `$SOURCE_CODE_DIR`,生成 60 页规范 PDF。

3. **说明书鉴别材料** → 转 **manual-material** 技能
   处理 `$MANUAL_PATH`,引导调用 crabcode-office-suite 办公套件生成规范 PDF。

4. **一致性校验** → 转 **consistency-check** 技能
   核对名称/版本号/日期在各材料中逐字一致。**每当有材料更新就应重跑。**

5. **申请包生成** → 转 **package-build** 技能
   统一命名归档 + 生成自查对照表。

6. **在线填报指导** → 转 **filing-guide** 技能
   给出平台逐字段填报卡片与 2026 新版申请表提示。

交接时把已知参数一并说清(软件全称/版本号/源码目录等),子技能缺参数会自行追问。
一道工序完成后,回到本技能视角汇报进度、勾掉清单、决定下一步。

## 全局红线（贯穿所有子技能,反复守）

1. **一致性第一**:软件全称、版本号、开发完成日期在申请表/源码页眉/说明书三处**逐字一致**(含大小写、空格、有无"V")。这是最高频驳回原因。
2. **真实不注水**:绝不伪造、PS、虚构;不用空行/纯注释/模板凑行数页数。
3. **日期自洽**:开发完成日期 ≤ 首次发表日期;企业申请不早于成立日期。
4. **2026 新版申请表**(以平台实际为准,非《办法》条文):经办人本人签字+身份证号、手抄"未使用 AI"诚信承诺、功能说明扩写至 500–1300 字、失信/征信惩戒。详见 GUIDE.md §8。
5. **隐私**:产物中不留身份证号等,由用户自行填入平台。
6. **免费**:官方普通登记自 2017-04-01 起免费,提醒用户勿被"官费"话术误导(代理费另计)。

## 交付

每完成一个阶段,向用户输出:已完成项、产物路径、待办清单(✅/❌/⚠️)、以及下一步建议。
全流程结束时提示:申请表须在平台自行生成填报,身份证明由用户放入,经办人须本人签字。
