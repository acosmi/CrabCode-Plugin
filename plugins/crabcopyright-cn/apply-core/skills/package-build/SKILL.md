# 软著申请包生成

把已完成的各项材料统一命名、归入一个规范目录,并产出一份**材料自查对照表**——
逐条对照官网规范标注是否达标,让用户提交前一眼看清有没有缺漏。**先读**
`${CRABCODE_PLUGIN_ROOT}/apply-core/GUIDE.md`(用其 §9 红线清单作为对照表的检查项来源)。

## 步骤

1. 创建输出目录 `软著申请-${SOFTWARE_NAME}${VERSION}/`。
2. 按命名规范拷入已有材料(缺的留占位并在对照表标 ❌):
   - `01-软件著作权登记申请表.pdf`（平台生成后放入）
   - `02-源代码鉴别材料.pdf`
   - `03-说明书鉴别材料.pdf`
   - `04-身份证明文件.pdf`（用户自备）
   - `05-其他材料/`（合作/委托/许可证明等补充件）
3. 生成 `材料自查对照表.md`。

## 材料自查对照表模板

```
# 软著申请材料自查对照表 · ${SOFTWARE_NAME} ${VERSION}

## 一、源代码鉴别材料
- [✅/❌] 共 60 页（或不足60页已交全部并标注总行数）
- [✅/❌] 每页 ≥50 行
- [✅/❌] 页眉含"软件全称 + 版本号",与申请表一致
- [✅/❌] 右上角连续页码
- [✅/❌] 首页=模块开头,末页=自然结尾,无注水

## 二、说明书鉴别材料
- [✅/❌] 含封面/目录/概述/功能说明(配截图)/运行环境
- [✅/❌] 每页 ≥30 行（截图页除外）
- [✅/❌] 功能描述与源代码对应
- [✅/❌] 名称、版本号与申请表一致

## 三、申请表与证明文件
- [✅/❌] 身份证明齐备且与著作权人一致
- [✅/❌] 特殊情形补充材料齐备（合作/委托/二次开发许可证明等）
- [✅/❌] 日期逻辑正确（开发完成 ≤ 首次发表）

## 四、提交前提醒
- 2026 新版申请表:经办人本人签字+身份证号、手抄"未使用 AI"承诺、功能说明 500–1300 字
- 官方普通登记免费
```

## 成功标准

- [ ] 输出目录含全部应有文件(缺项在对照表标 ❌)
- [ ] 对照表涵盖源代码、说明书、申请表三大类的检查项
- [ ] 存在 ❌ 项时在终端醒目提示,不假装完成

## 检查点

若对照表出现 ❌,不要宣称"打包完成即可提交",而应回到软著申请管家补齐对应材料。
