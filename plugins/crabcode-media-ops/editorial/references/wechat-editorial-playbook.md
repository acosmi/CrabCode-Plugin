# 微信原创观点编辑手册

- 标题承诺一个具体矛盾或判断，不用“震惊、必看、颠覆”等廉价强度。
- 开头在三段内交代事件、冲突和本文判断。
- 每一节只推进一个问题；事实之后解释“为什么”，再提出作者推论。
- 至少处理一个最强反方，说明结论在什么条件下不成立。
- 句长和段长有变化；移动端避免连续大段。
- 结尾不是机械总结，而是把判断落到普通人的选择、成本或行动。
- 不编造第一人称体验。没有真实材料时用“从公开资料看”“本文判断”。
- AI 披露采用实际平台功能、正文或文件元数据中的已确认方式，不固定成唯一文案。
- 写作上下文不得包含第三方参考原文、附件或原路径；只使用结构化研究包、referenceId 和不可复制清单，在 fresh context 中独立建立中心判断、案例组合和论证顺序。
- 禁止按参考文逐段映射、保持相同论证推进后换同义词，或复用标志性标题结构、比喻与结尾。无法证明独立时回到选题与研究重新构思。
