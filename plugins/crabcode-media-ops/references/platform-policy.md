# 平台发布/互动政策边界与本插件运营原则

本文件汇总各主流内容平台的发布与互动政策边界（用于内容合规判断与发布前检查），并给出本插件统一遵循的人工确认、限流、审计原则。具体平台规则随时间变化，**以平台官方最新规范与 `mediaops.policy_status` / `mediaops.readiness.inspect` 返回为权威**，本文件用于运营判断与风险提示。

## 通用红线（所有平台）
- 不发布违法违规、虚假信息、人身攻击、侵权（图文/字体/音乐）内容。
- 涉医疗、金融、法律、保健等专业领域内容须有资质或可信来源，避免诊断/荐股式表述。
- 标题不得标题党、不得与正文不符；不得诱导分享/关注的违规话术。
- AI 生成/辅助内容须按《人工智能生成合成内容标识办法》保留显式标识（见 `ai-labeling-compliance.md`）。

## 微信公众号
- 内容类型：图文消息（news）与图片消息（newspic），按形态择一。
- 标题与封面须真实反映内容；外链跳转受限。
- 互动：留言需开通且经精选机制，回复有频次与审核约束。
- 发布：草稿箱（draft）→ 群发/发布（freepublish）需账号权限与凭证（**Gate B**）。

## 小红书
- 图文笔记以图为主，避免硬广导流、违规引流（私域/微信号）话术。
- 话题标签需相关，不堆砌；首图与标题不得夸大。
- 互动：评论回复注意社区氛围，避免营销味过重被限流。

## 今日头条
- 信息流分发，首段与标题影响推荐；禁标题党与低质拼凑。
- 配图须有版权来源；时政、社会类内容审核严格。

## 抖音 / 微博 / B站（视频/短内容，本阶段仅规划）
- 真发布 API 与互动发送均属 **Gate B**，未配凭证前不开放。
- 内容须符合各平台社区规范与版权要求。

## 本插件运营原则
1. **人工确认 gate**：任何发布与评论发送前必须经人工审批（`mediaops.approval.request`），未获批不打发布包、不发送。
2. **限流与节奏**：遵循 `mediaops.policy_status` 给出的限流边界；批量操作分批，避免触发平台风控。
3. **审计留痕**：每条内容保留 AI 辅助痕迹、来源清单、人工确认记录，经 `mediaops.publish.history` 回查。
4. **Gate 分级**：Gate A 仅创作闭环 + 发布包（零凭证）；真平台 API、浏览器辅助、评论发送为 Gate B，需平台凭证配置后开放。
