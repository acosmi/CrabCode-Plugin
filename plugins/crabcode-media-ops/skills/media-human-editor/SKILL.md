# media-human-editor — 人味编辑（编辑质量提升）

## 这是什么，不是什么（合规红线先读）
- **是**：把初稿当成"编辑提质"对象，用专业编辑手法提升真实质感、可读性、可信度。
- **不是**：抹掉/伪造/隐匿 AI 标识，**不是**为了规避 AI 检测。AI 辅助内容发布前**必须**保留显式 AI 辅助标识（见 `references/ai-labeling-compliance.md`）。任何"去味=去标识"或"规避检测"的诉求，明确拒绝并解释合规要求。

"去 AI 味"的正确含义 = 提高人工编辑质量，让内容更专业、更具体、更有依据，而非掩盖其 AI 辅助来源。

## 五层编辑框架（逐层过）

### 1. Brand Voice（品牌口吻）
- 对照品牌 profile（人设/口吻/禁用词/栏目/受众，`references/brand-profile-schema.md`）。
- 统一人称、语气、术语；剔除禁用词；让稿子像"这个号"会说的话。

### 2. Evidence Grounding（证据落地）
- 每个关键论断配可追溯来源；把"据说/普遍认为"换成具体出处或删除。
- 数字、案例、引语具体化；空泛断言换成可验证事实。

### 3. Anti-template（反模板）
- 扫描并改写模板化信号（详见 `references/humanize-rules.md`）：
  - 万能开头（"在当今这个……的时代"）
  - 三段式套路（总分总八股、"首先/其次/最后"硬框）
  - 空泛排比（连续排比但无信息增量）
  - 过度转折（"然而/但是"堆叠）
  - 滥用破折号/分号制造伪深度
  - 结尾空泛升华（"让我们一起……"）
  - 形容词堆砌、AI 高频套话

### 4. Human Detail（人的细节）
- 加入具体场景、真实数据、第一手观察、有立场的判断。
- 适度的不完美与个性表达（合适的口语、具体比喻），避免通篇平滑无棱角。

### 5. Editor Review（终审）
- 通读检查逻辑连贯、信息密度、节奏；删冗余、补缺口。
- 确认事实核查结论已采纳；确认 AI 辅助标识保留在成稿末尾。
- 配合 `mediaops.readiness.inspect` 做确定性校验收尾。

## 反模板信号速查清单
万能开头 / 三段式 / 空泛排比 / 过度转折 / 滥用破折号 / 结尾升华套话 / 形容词堆砌 / "不仅……而且……"滥用 / 伪权威（"研究表明"无出处） / 通篇中性无观点 / 段落长度过于均匀。命中即改写，改写要补信息而非仅换词。
