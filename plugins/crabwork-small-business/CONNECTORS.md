# 连接器

## 工具引用机制

插件文件用 `~~类别` 作为占位符,代表用户在该类别下连接的具体工具。例如 `~~accounting` 可指 QuickBooks 或任何提供 MCP 服务的财务系统。

插件是**工具无关**的——它们以类别(财务、收款、CRM、设计等)而非具体产品来描述工作流。`.mcp.json` 预配置了具体的 MCP 服务,但同类别下的任何 MCP 服务都可使用。

## 本插件的连接器

| 类别 | 内置服务 | 启用能力 |
|---|---|---|
| 财务 / 记账 | QuickBooks | 现金预测、毛利、月末结账、报税准备 |
| 收款 | PayPal、Stripe、Square | 交易、发票、争议、退款、POS 数据 |
| CRM / 营销 | HubSpot | 线索、商机、营销活动、客服工单 |
| 设计 | Canva | 社媒与邮件素材生成 |
| 电子签 | DocuSign | 合同审阅、录用通知签署 |
| 邮件 | Gmail | 邮件草拟、工单处理、合同来源 |
| 日历 | Google Calendar | 会议准备、电话时段、每周事项 |
| 文件存储 | Google Drive | 模板、文件、导出物 |
| 聊天 | Slack | 简报推送与通知 |

> 提示:`.mcp.json` 中 `url` 为空的连接器(若有)保留原样,连接后即可启用对应能力;同类别下你也可替换为自己偏好的 MCP 服务。
