# 线索打分 —— 线索分诊(lead-triage)

打分维度、权重与 ICP 默认值,以及**中性字段 → 各 CRM 平台**的映射。打分维度(互动 / 匹配 / 紧迫 / 近期触达)与 CRM 无关;**具体字段名、对象与互动信号来源以你实际连接的 CRM 当年开放平台文档为准**。

> **最小必要提示(《个人信息保护法》):** 下列字段是实现"排序跟进"目的所必需的最小集合——只拉取这些,不为打分额外收集无关个人信息。所用联系人信息须为**合法取得**(告知同意 / 合同必需等)。

---

## 要拉取的字段(中性概念 → 平台映射)

技能只谈中性概念;下表右列给出各平台对应对象/字段(以该平台当年官方文档为准)。小微通常只连一家,按已连接的那家取数。

| 中性概念 | 用途 | HubSpot(外贸可选)字段 | 企业微信 / 钉钉 / 飞书 / 有赞 对应 |
|---|---|---|---|
| 姓名 | 显示 | `firstname`、`lastname` | 外部联系人姓名 / 成员;有赞:会员昵称 |
| 公司 | 显示 + ICP 匹配 | `company` | 联系人所属企业 / 客户标签;有赞常无 |
| 联系方式 | 跟进草稿收件人 | `email` | 微信 / 手机号 / 会员账号(**属个人信息,最小必要**) |
| 线索阶段 | 过滤(保留潜在客户 / 已验证) | `lifecyclestage`、`hs_lead_status` | SCRM 阶段 / 标签;有赞:会员分层;企微无标准"生命周期",用标签近似 |
| 行业 / 规模 | 公司匹配 | `industry`、`numemployees` | 客户标签 / 自定义字段;常为空→按未知处理 |
| 创建时间 | 紧迫度——线索存续时长 | `createdate` | 加好友 / 建档 / 首单时间 |
| 最后活动 | 近期触达扣分 | `hs_last_activity_date` | 最近互动 / 消息 / 下单时间 |
| 备注新鲜度 | 紧迫度 | `notes_last_updated` | 跟进备注 / SCRM 动态 |
| 互动信号 | 互动打分(见下) | `hs_sales_email_last_replied`、`hs_email_open`、`hs_analytics_last_visit_timestamp`、`num_contacted_notes` | 见「互动」维度的平台信号映射 |

**企业微信无"商机管道"概念**:打分落到**互动与 SCRM 标签**,而非管道阶段金额。**有赞**以**会员 / 订单**为主,"线索"近似"加购未付 / 咨询未下单"。字段不存在时按未知处理并据此打分,不臆造。

---

## 打分模型(0–100 综合分)

四个维度,各 0–25,求和为综合分。**维度与权重跨平台不变**;各平台把信号映射到等价物。

### 互动(0–25)
仅计入近 30 天的信号,更早的记 0。

| 信号 | 分值 |
|---|---|
| 近 14 天内有回复(邮件回复 / 微信回复 / 客服咨询) | +15 |
| 近 7 天内有打开 / 浏览(邮件打开 / 朋友圈或商品浏览,无回复) | +8 |
| 近 7 天内有主动触点(网站访问 / 进店 / 加购) | +5 |
| 触达 >3 次仍无回复 | −5 |

> **信号来源随平台而定(等价映射):** HubSpot——邮件回复 / 打开 / 网站访问;**企业微信**——客户消息回复 / 客户朋友圈互动 / 客户群活跃 / SCRM 标签;**有赞**——下单 / 复购 / 加购 / 客服咨询;**钉钉 / 飞书**——套件内 CRM 记录的跟进与互动。取该平台能提供的等价信号,取不到的记 0,不臆造。

### 公司匹配(0–25)
业主未指定时的默认 ICP:不限行业,1–50 人。

| 匹配 | 分值 |
|---|---|
| 行业 + 规模均命中 ICP | 25 |
| 规模命中,行业未知 | 15 |
| 行业命中,规模未知 | 12 |
| 都不命中或都未知 | 5 |

### 紧迫度(0–25)

| 信号 | 分值 |
|---|---|
| 备注含紧迫措辞("急""尽快""截止""预算已批"等) | +15 |
| 线索存续 7–21 天(黄金跟进窗口) | +10 |
| 线索存续 <7 天 | +5 |
| 线索存续 >60 天 | −5 |

### 近期触达扣分(从综合分中扣减)

| 最后活动 | 扣减 |
|---|---|
| 24 小时内 | −25 |
| 1–3 天前 | −10 |
| 4–7 天前 | −5 |
| >7 天前 | 0 |

---

## 自定义 ICP(运行时覆盖)

若业主在运行时指定 ICP(如"聚焦餐饮,10–100 人"),仅对本次会话生效。

---

## 自动化决策提示(《个人信息保护法》第 24 条)

本模型输出的是**内部跟进优先级**,属自动化决策。分数不得用于对客户在成交价格等交易条件上作差别待遇(不做价格歧视 / 大数据杀熟);业主应保留人工干预空间,可随时人工调整排序或剔除条目。SCRM(企业微信)涉客户朋友圈 / 客户群,个人信息面更广,护栏只增不减。深度合规评估移交 `crablaw-cn:data-activity-triage`。
