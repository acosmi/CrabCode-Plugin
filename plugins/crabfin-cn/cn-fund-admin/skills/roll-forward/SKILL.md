# 基金滚存

生成基金期间滚存表，追踪NAV从期初到期末的完整变动路径。

> **科目依据**：`sources/mappings/基金会计科目体系_v1.csv`
> **估值指引**：`sources/laws/AMAC基金估值指引.md`（CAS 39三层公允价值）
> **份额登记**：`sources/laws/份额登记确认规则.md`（T日交易/T+1确认/T+2可查询）

## 操作步骤

### Step 1 — 滚存计算

```
期初NAV
  + 投资者申购/追加
  − 投资者赎回/分配返还
  + 投资收益(已实现+未实现)
  + 公允价值变动
  + 利息/股息收入
  − 管理费/托管费/业绩报酬/其他费用
  − 基金分红/分配
  = 期末NAV
```

### Step 2 — 输出滚存表

| 项目 | 金额(万) | 备注 |
|------|---------|------|
| **期初NAV** | | |
| **资本变动** | | TA申赎明细 |
| **投资损益** | | L1+L2/L3重估 |
| **费用** | | 计提表 |
| **分配** | | 分红公告 |
| **期末NAV** | | 与估值表核对 |

### Step 3 — 关键校验

| 校验项 | 方法 |
|--------|------|
| 期末NAV一致性 | 滚存 = 估值表NAV |
| 申赎净额 | = TA当期净申购 |
| 公允价值变动 | 期末市值 − (期初成本 + 买入 − 卖出摊销) |
| 费用合计 | = `accrual-schedule` 合计数 |

### Step 4 — 分红处理

**除息日**：NAV下折 → 借:未分配利润 / 贷:应付分红款
**发放日**：借:应付分红款 / 贷:银行存款或实收基金（份额确认：T+1确认/T+2可查询）

**数据不可得fallback**：TA数据不可得 → 以托管行数据暂代，标注"待TA确认"。

### 合规声明

```
⚠️ 期末NAV须与估值表一致。分红份额登记按T+1确认/T+2可查询规则处理。
公允价值变动须按L1/L2/L3分别列示，L3资产估值假设变更单独标注。
```

## 协作接口

| fin-core子技能 | 何时调用 | 期望取回的数据 |
|---------------|---------|--------------|
| `accrual-schedule` | 费用填入 | 费用计提明细 |
| `nav-tieout` | 期末NAV验证 | 估值表NAV |
| `gl-recon` | 申赎TA数据 | 申赎流水 |
