# 基金差异追溯

对基金核算中出现的差异逐层追溯定位，按五类差异分类框架。

> **估值依据**：`sources/mappings/估值层级判定_v1.md`（L1/L2/L3分层/参数）
> **AMAC指引**：`sources/laws/AMAC基金估值指引.md`
> **份额登记**：`sources/laws/份额登记确认规则.md`（T+1确认/T+2可查询/差错处理）

## 操作步骤

### Step 1 — 差异发现与分类

**输入**：对账差异数据
**输出**：差异归入以下五类之一

| 类别 | 定义 | 常见原因 |
|------|------|---------|
| 估值差异(Valuation) | 同一资产不同估值方法/参数 | L1/L2/L3层级判断差异、流动性折扣参数 |
| 入账差异(Booking) | 同一交易不同系统记录时点/金额不同 | 交易日vs交割日、汇率取价 |
| 计算差异(Calculation) | 公式/费率/基数不同 | 费率变更未同步、计提口径不同 |
| 数据差异(Data) | 底层资产数据源不同 | 行情源差异、公司行为数据 |
| 份额登记差异(Registration) | 份额登记端数量与托管端不一致 | T日确认延迟、份额余额不符（见 `份额登记确认规则.md` 第四章） |

### Step 2 — 逐层追溯

```
发现差异 → 确认金额和方向(多计/少计)
  → 定位差异归属(头寸?费用?估值?汇率?)
  → 逐层拆解：
    估值差异→检查层级(L1/L2/L3)和参数
    费率差异→检查费率标准和基数
    交易差异→检查交易日/交割日和汇率
    份额差异→检查TA确认时点(应按T+1确认)
  → 确定责任方(管理人/托管行/外包服务商)
  → 输出追溯报告+建议调整分录
```

### Step 3 — 估值差异专项

- L1资产差异 → 检查行情源（如上交所vs深交所）
- L2资产差异 → 检查可比公司选择/乘数选取
- L3资产差异 → 检查DCF参数（折现率/增长率）和流动性折扣

### Step 4 — 输出差异追溯报告

```
差异发现日期/金额/占NAV比例
差异定位：类别/涉及资产/管理人估值 vs 托管行估值/根因
建议调整分录
待跟进事项
```

### 合规声明

```
⚠️ 差异追溯结论须经托管行确认后方可作为调整依据。
NAV差异超过合同约定容忍度（通常0.05%-0.1%）不得签发NAV公告。
```

## 协作接口

| fin-core子技能 | 何时调用 | 期望取回的数据 |
|---------------|---------|--------------|
| `gl-recon` | 差异来自对账发现 | 对账差异清单 |
| `nav-tieout` | 估值差异需验证 | 估值表/总账/TA三方NAV |
| `variance-commentary` | 重大差异需管理层说明 | 差异说明模板 |
