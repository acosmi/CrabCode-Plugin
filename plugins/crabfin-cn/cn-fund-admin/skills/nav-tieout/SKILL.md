# 基金NAV勾稽

对基金NAV进行跨系统一致性校验：估值表NAV = 总账NAV = TA份额净值。

> **估值依据**：`sources/laws/AMAC基金估值指引.md`
> **底稿模板**：`sources/templates/NAV勾稽底稿_脱敏模板.md`

## 操作步骤

### Step 1 — 估值表NAV计算验证

| 检查项 | 方法 |
|--------|------|
| 资产总值 | Σ(持仓数量 × 单位估值) + 应收 + 银行存款 |
| 负债总值 | Σ应付管理费+托管费+业绩报酬+交易费用+其他 |
| 估值方法 | L1收盘价 / L2可观察输入 / L3检查参数 |
| 汇率 | T日人民币中间价 |

### Step 2 — 估值表NAV vs 总账NAV

```
估值表NAV + 未入账调整 − 重复计算 = 总账NAV
差异须 = 0 或可合理解释
```

### Step 3 — NAV vs TA份额净值

```
NAV ÷ 份额总数 = 单位净值 vs TA系统单位净值
差异须 < 0.0001
```

### Step 4 — 输出勾稽底稿

| 项目 | 金额(万) | 来源 |
|------|---------|------|
| 交易性金融资产(L1/L2/L3) | | 估值表 |
| 银行存款/结算备付金 | | 银行/券商对账单 |
| 应收利息/股利/申购款 | | |
| 应付管理费/托管费/业绩报酬 | | 计提表 |
| **NAV** | | |
| 份额总数(万份) | | TA |
| **单位净值** | | |

### Step 5 — 差异处理

| 差异项 | 核查方向 |
|--------|---------|
| 估值表≠总账 | 未过账凭证/费用计提差异 |
| NAV/份额≠TA净值 | 份额总数滞后/分红除息 |
| 管理人≠托管行 | 走 `break-trace` |

### 合规声明

```
⚠️ NAV勾稽差异超过合同约定容忍度（通常NAV的0.05%-0.1%），不得签发NAV公告。
须先行完成差异追溯和调整。勾稽底稿须编制人+复核人签核。
```

## 协作接口

| fin-core子技能 | 何时调用 | 期望取回的数据 |
|---------------|---------|--------------|
| `break-trace` | 差异超容忍度 | 差异根因/调整分录 |

| `accrual-schedule` | 费用验证 | 费用计提明细 |
