# 基金总账对账

对基金总账与托管行、TA、交易系统进行四维对账。

> **科目依据**：`sources/mappings/基金会计科目体系_v1.csv`（中国基金会计科目标准）
> **份额登记**：`sources/laws/份额登记确认规则.md`（份额变动须逐笔匹配托管行/差错处理）

## 操作步骤

### Step 1 — 确定对账维度

| 对账方 | 内容 | 频率 |
|--------|------|------|
| 管理人 vs 托管行 | NAV/资产持仓/费用/银行存款 | 每日/每估值日 |
| 管理人 vs TA | 份额余额/申赎流水/分红 | 每日 |
| 管理人 vs 交易系统 | 持仓数量成本/交易流水 | 每日/每交易日 |
| 总账 vs 子账 | 科目余额一致性 | 每月 |

### Step 2 — 科目对账

| 资产类别 | 对账要点 |
|---------|---------|
| 银行存款 | 日终余额/未达账项/受限资金 |
| 结算备付金 | 交易用/清算用/利息计提 |
| 交易性金融资产 | 持仓数量/成本均价/市值/估值方法 |
| 应收申购款/应付赎回款 | 交易确认日期/交收日期 |
| 应收利息/股利 | 计提日期/到账 |
| 管理费/托管费应付 | 费率/计提基数/金额 |

### Step 3 — 底稿与差异处理

| 差异类型 | 处理方式 |
|---------|---------|
| 未达账项 | 编制银行余额调节表，合理差异 |
| 估值差异 | 走 `break-trace` 流程 |
| 汇率差异 | 确认双方取价时点和汇率源 |
| 费用计提差异 | 检查费率标准/基数/起始日 |
| 申赎确认差异 | 确认TA数据截止时点 |

### Step 4 — 输出对账结论

```
对账期间/维度
✅ 一致项目
⚠️ 容忍差异（未达账项等）
❌ 须跟进差异 → 行动计划/负责人/截止日
```

### 合规声明

```
⚠️ 份额变动数据须与托管银行份额数据逐笔匹配，差异须走差错处理流程。
对账记录须经编制人+复核人签字，保存期限满足监管要求。
```

## 协作接口

| fin-core子技能 | 何时调用 | 期望取回的数据 |
|---------------|---------|--------------|
| `break-trace` | 发现差异需追溯 | 差异定位/根因 |
| `nav-tieout` | NAV差异验证 | 估值表/总账/TA三方NAV |
| `roll-forward` | 科目余额变动 | 滚存表 |
