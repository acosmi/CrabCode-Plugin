# 单位经济模型

基于中国私募股权投资实践，按行业分类构建标的企业的单位经济模型（Unit Economics），帮助投资团队穿透企业整体财务数据，在最小业务单元层面验证商业模式的健康度。

## 一、单位经济模型核心指标

### 1.1 通用指标

| 指标 | 定义 | 健康基准 |
|------|------|---------|
| **Unit Revenue** | 单位收入（如ARPU/客单价/单笔交易收入） | — |
| **Unit Cost** | 单位成本（变动成本） | — |
| **Unit Gross Profit** | 单位毛利 = Unit Revenue − Unit Cost | > 0 |
| **Gross Margin** | 毛利率 = Unit Gross Profit / Unit Revenue | 因行业而异 |
| **CAC**（获客成本）| 总销售与市场费用 / 新增客户数 | — |
| **LTV**（客户生命周期价值）| Unit Gross Profit × 客户平均生命周期（月/年） | — |
| **LTV/CAC** | 客户价值与获客成本比 | > **3x**（续费型业务）/ > **1.5x**（交易型业务） |
| **CAC Payback**（获客回收期）| CAC / 月均Unit Gross Profit | < **12个月**（SaaS）/ < **6个月**（消费） |

### 1.2 续费型业务特殊指标
- **Retention Rate**（留存率）：年度净收入留存率（NRR）> **100%** 为优秀
- **Churn Rate**（流失率）：月度流失率 < **2%**（SMB）/ < **1%**（Enterprise）

## 二、按行业构建单位经济模型

依据 `sources/mappings/尽调必查清单_v1.csv` 中业务尽调和财务尽调子项（收入确认政策、毛利率、客户/供应商依赖度），按行业类型搭建差异化的单位经济模型。

### 2.1 SaaS/软件订阅

```
单位经济单元 = 单个客户/单个坐席
Unit Revenue = ARPU（月均客户收入）
Unit Cost = 云基础设施成本 + 客户支持成本 + 第三方软件许可费
CAC = （销售团队薪酬 + 市场推广费用）/ 同期新增客户数
LTV = ARPU × Gross Margin% × 平均客户生命周期（月）
```

关键验证点：
- NRR（净收入留存率）：是否 > **100%**（upsell弥补流失）
- LTV/CAC：是否 > **3x**
- CAC Payback：是否 < **12个月**
- 大客户vs小客户的单位经济差异（Enterprise vs SMB）

### 2.2 电商/消费品牌

```
单位经济单元 = 单笔订单/单个客户
Unit Revenue = 客单价（AOV）
Unit Cost = 商品成本（COGS）+ 包装物流成本 + 支付手续费
CAC = 营销费用 / 同期新增客户数（或首单新客数）
LTV = （客单价 × 毛利率）× 复购次数
```

关键验证点：
- 首单盈利性：首单毛利是否 > CAC（首单盈利为正才是健康的DTC模型）
- 复购率：季度复购率 > **20%** 为良好
- 客单价趋势：是否在增长（品牌溢价能力）
- 退货/退款率对单位经济的影响

### 2.3 交易平台/双边市场

```
单位经济单元 = 单笔交易
Unit Revenue = 交易佣金/手续费（Take Rate% × GMV）
Unit Cost = 支付通道费 + 风控成本 + 客服成本
CAC（需求端）= 用户补贴 + 营销费用 / 新增交易用户
CAC（供给端）= 商家/司机/房东入驻激励
LTV = 单用户月均交易次数 × Take Rate × 用户平均留存月数
```

关键验证点：
- Take Rate 趋势：是否稳定或提升
- 双边CAC 合计 vs 单用户毛利：是否为正
- 补贴依赖度：剔除补贴后的真实LTV/CAC
- GMV到收入的转化率（take rate是否可持续）

### 2.4 硬件/智能制造

```
单位经济单元 = 单台设备/单个SKU
Unit Revenue = 出厂价（ASP）
Unit Cost = BOM成本 + 直接人工 + 制造费用分摊
Unit Gross Profit = ASP − Unit Cost
```

关键验证点：
- BOM成本中核心零部件的供应商依赖度和价格波动风险
- 规模效应对Unit Cost的影响（产量翻倍，单位成本下降幅度）
- 不同SKU/型号的单位毛利率差异（高端 vs 低端产品线）
- 售后服务/维保收入的单位经济（如有）

### 2.5 生物医药/创新药

```
单位经济单元 = 单个药品适应症/单个患者年用药
Unit Revenue = 年治疗费用（根据定价策略和医保谈判结果估算）
Unit Cost = API/原料成本 + 生产成本 + 分销成本
患者规模 = 目标适应症患病人数 × 诊断率 × 治疗率 × 市场份额
```

关键验证点：
- 医保谈判对定价的影响（集采降价幅度预估）
- 竞品上市时间窗口（专利独占期还剩几年）
- 产能建设的资本开支节奏
- 不同适应症之间的单位经济差异

## 三、盈利拐点分析

### 3.1 盈亏平衡点计算

```
月度盈亏平衡客户数 = 月度固定成本 / 单位客户月度毛利
```

- 固定成本：研发团队薪酬、行政费用、办公租金等
- 单位客户月度毛利 = ARPU（月）× Gross Margin%

### 3.2 规模效应评估

| 规模阶段 | 典型特征 | 关键观测点 |
|----------|---------|-----------|
| 早期（< 1000客户） | Unit Cost 高，CAC 高 | Unit Cost 随规模下降的速率 |
| 成长期（1000-10000客户） | Unit Cost 快速下降，CAC 企稳 | 毛利率提升拐点是否出现 |
| 成熟期（> 10000客户） | 规模效应递减，边际改善有限 | 是否存在自然垄断性壁垒 |

## 四、行业基准参考

以下行业基准来自市场惯例（非监管规定），供投资判断参考：

| 行业 | 健康毛利率 | 健康LTV/CAC | 典型盈利拐点 |
|------|-----------|------------|-------------|
| SaaS/软件 | > 70% | > 3x | 12-24个月 |
| 电商/DTC | > 40%（品牌）/ > 20%（平台型） | > 1.5x | 6-12个月 |
| 交易平台 | > 50%（take rate口径） | > 2x | 18-36个月 |
| 硬件/制造 | > 25% | N/A | 依产能利用率而定 |
| 生物医药 | > 80%（创新药） | N/A | 5-10年（从IND到商业化） |

## 五、单位经济与估值的联动

单位经济模型的结论应直接影响估值参数：
- LTV/CAC < **2x** → 估值倍数应低于可比公司
- CAC Payback > **18个月** → 增长质量存疑，须对增长预期打折
- 毛利率持续下滑 → 竞争加剧信号，终端估值下行
- NRR < **100%** → 续费型业务的核心价值受损，须重新评估

## 六、输出格式

1. **行业单位经济模型定义**：明确什么是"一个单位"
2. **单位经济核心指标表**（含实际值、可比公司基准、健康线）
3. **LTV/CAC 及 CAC Payback 分析**
4. **盈利拐点预测**：何时达到盈亏平衡
5. **规模效应路线图**：关键假设下单位成本随规模下降的路径
6. **Red Flag 清单**：单位经济层面发现的问题（如首单亏损、获客成本走高、毛利率持续下滑）

> 单位经济模型分析的目的是验证商业模式的"原子级"健康度。一个企业可以在整体层面盈利但单位经济恶化（通过削减研发/行政费用掩盖），也可能整体亏损但单位经济健康（处于投入期）。投资判断必须基于单位经济而非整体利润表。
