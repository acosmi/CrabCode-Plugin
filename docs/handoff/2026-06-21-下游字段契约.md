## 字段二:`groups`

仅出现在「伞形工作流」条目上(当前 **3** 个),用于驱动卡片「按板块分组显示」。结构为**有序数组**,每项三字段:

```json
"groups": [
  {
    "name":        "fin-core",                   // kebab-case 稳定 id(程序键,勿展示)
    "displayName": "CrabFin 财务建模底座",         // 中文板块标题(给用户看)
    "skills": ["audit-xls", "dcf-model", "..."]  // 该板块下的 skill 目录名(裸名)
  }
]
```

当前带 `groups` 的条目:

| 条目 | 组数 | skill 数 |
|---|---|---|
| crablaw-cn | 11 | 86 |
| crabfin-cn | 7 | 54 |
| crabwork-small-business | 11 | 31 |

**保证(可作为下游断言/校验):**

- `groups[].skills` 与该插件磁盘上的 skill 目录**完美双射**——无幽灵、无孤儿、无重复。
- 同一条目内 skill 不跨组重复。
- `groups` 顺序即建议展示顺序。

---

## 渲染规则

1. `tier=workflow` 且**有** `groups` → 工作流插槽内,卡片按 `groups` 分组,每组用 `displayName` 作小标题,组内列 `skills` 卡片。不要求一屏,可纵向滚动。
2. `tier=workflow` 且**无** `groups` → 卡片平铺(条目少时合理,如 agent-sdk-dev)。
3. `tier=connector` → 连接器区。
4. `tier=capability` → 独立能力卡片区。
5. `name` 是程序键不展示;展示用 `displayName` / `shortDescription` / `brandColor` 等既有字段。
