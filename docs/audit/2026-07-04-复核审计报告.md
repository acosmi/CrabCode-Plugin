## 三、FAIL — 问题项

### 3.1 品牌违规（高）—— `lint:brand` 30+ 处命中

| 类型 | 数量 | 示例路径 |
|------|------|----------|
| `.mcp.json` 含 `claude` 域名 | 8 | `plugins/crabwork-bio-research/.mcp.json` (pubmed.mcp.claude.com 等) |
| `.mcp.json` 含 `anthropic` 端点 | 5 | `plugins/crabwork-sales/.mcp.json` (mcp.hubspot.com/anthropic) |
| `README.md` 含 `anthropic` 声明 | 14 | 各 crabwork-* 插件 README |
| `genomes.yaml` 含 `opus`（假阳性） | 1 | `plugins/crabwork-bio-research/skills/.../genomes.yaml:102` |

**根因**：上游衍生插件的 MCP URL 和 README 属性声明未完全清洗。`.mcp.json` 域名是第三方服务端点；README 应根据 CRABCODE.md 安全规则仅将 upstream 引用放在 legal notices 中。

### 3.2 测试：品牌测试预期失效（高）

`tests/analysis.test.ts` → `brand guard passes repository product files` 预期为干净但实际命中违规行导致该测试失败。

### 3.3 缺失 README.md —— 9 个插件（中）

| 插件 |
|------|
| ai-api-dev |

| cn-legal-study |

### 3.4 TypeScript 编译不可用（中）

`bun run typecheck` → `tsc: command not found`。`typescript` 在 `devDependencies` 中但环境未安装 `node_modules`，CI 需 `bun install` 前置。

### 3.5 陈旧构建产物（低）

`dist/cli.js` 已提交至仓库，存在源码与构建版本不一致的风险。

## 四、修复建议

### P0 — 品牌清理

1. **`.mcp.json` 域名**：将 5 个文件中的 `mcp.claude.com` / `mcp.hubspot.com/anthropic` URL 替换为国内代理设施或 CrabCode 兼容端点，或迁移至 `CONNECTORS.md` 并在这 5 个 mcp.json 中标注待替换
2. **README 属性声明**：14 个 README.md 中的 "anthropics/knowledge-work-plugins" → 移入 `docs/legal/` 或精简为 "upstream"
3. **假阳性 `opus`**：`genomes.yaml:102` 的 `"Xenopus tropicalis"` 说明字段含 `opus` 是物种名假阳性，加入 brand scan ignore

### P1 — 基础设施

4. 补充 9 个缺失 README.md
5. `typecheck` 脚本改为 `bun x --no-install tsc --noEmit` 或确保 CI 中 `bun install` 前置
6. 将 `dist/` 加入 `.gitignore`（仅 CI 产出）
