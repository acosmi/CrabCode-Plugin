# legal / finance 通用版参考优化建议(对照上游,优化现有中国版)

- **日期**: 2026-06-21
- **性质**: 建议清单,**不修改任何插件**(crablaw-cn / crabfin-cn 保持原样)
- **方法**: 对照上游 `anthropics/knowledge-work-plugins` 的 `legal`(9 skill)、`finance`(8 skill)的能力骨架,审视现有中国版的通用能力缺口
- **重要前提**: 上游为美国法/US GAAP 通用骨架,**仅作流程骨架参考**;任何并入都必须按中国法规(《民法典》《电子签名法》《企业内部控制基本规范》《企业会计准则 CAS》等)本地化,**不可直接照搬**

## 一、legal:上游 9 skill 对现有 crablaw-cn(13 板块)的覆盖判定

| 上游 skill | 上游能力 | 现有中国版覆盖 | 缺口判定 | 借鉴建议 |
|---|---|---|---|---|
| `legal-risk-assessment` | 严重性×可能性矩阵分级 + 升级路径 | 各板块有分散的 risk-summary / risk-assessment / termination-risk-review,但**无统一分级框架** | **高价值缺口** | 在 `matter-core` 增设横切「法律风险分级框架(severity×likelihood + 升级阈值)」,供各业务板块复用,统一风险口径 |
| `close`/`brief` 类 `brief` | 每日/主题/事件法务简报 | cn-litigation 有 matter-briefing(单案件);**无跨案件每日简报** | 中价值缺口 | matter-core 可加「法务每日简报」(汇总待办期限+审查队列+新动态),与 crabwork-engineering 的 standup 思路一致 |
| `legal-response` | 模板化常见法律咨询应答 + 升级 | cn-legal-aid 有 plain-language-letter / client-comms-log;**无模板化咨询应答库** | 中价值缺口 | 可在 cn-legal-aid 或 matter-core 增设「常见法律咨询模板应答 + 超范围自动升级」,模板需按中国法本地化 |
| `meeting-briefing` | 法律会议结构化简报 + 行动项跟踪 | 无对应 | 中价值缺口 | 通用骨架可借鉴,落到 matter-core(会议准备+行动项闭环) |
| `signature-request` | 电子签名前检查 + 路由编排 | cn-corporate 有 closing-checklist(交割);**无通用签署编排** | 中价值缺口(需本地化) | 借鉴流程骨架,按《电子签名法》本地化(可靠电子签名要件、CA 证书),落到 cn-corporate 或 matter-core |
| `vendor-check` | 跨系统聚合供应商现有协议状态 | cn-corporate 有 material-contract-schedule | 部分覆盖 | 低优先;可在 material-contract-schedule 增加「跨来源供应商协议聚合」视角 |
| `review-contract` | 对照谈判 playbook 审合同 | cn-contract 有 review / clause-redraft / risk-summary,**覆盖强且更本地化** | 已覆盖 | 仅可借鉴「组织谈判底线库(playbook)」概念,增强 review 的一致性 |
| `triage-nda` | NDA 红黄绿快速分流 | cn-contract 有 nda-review | 已覆盖 | 可借鉴「GREEN/YELLOW/RED 分级分流」机制增强 nda-review 的决策清晰度 |
| `compliance-check` | 对任意业务倡议做通用合规检查 | cn-product(feature/launch)、cn-regulatory(gap)、cn-ai-governance 等**按领域拆分覆盖** | 已覆盖(更细) | 中国版按领域拆分更贴合监管;无需引入通用入口,避免与现有重叠 |

## 二、finance:上游 8 skill 对现有 crabfin-cn(7 板块)的覆盖判定

| 上游 skill | 上游能力 | 现有中国版覆盖 | 缺口判定 | 借鉴建议 |
|---|---|---|---|---|
| `close-management` | 月末结账流程编排(任务排序/依赖/状态) | cn-fund-admin 有 gl-recon / nav-tieout / roll-forward / accrual-schedule(**结账要素齐,但无统一编排器**) | **高价值缺口** | 在 cn-fund-admin 增设「月末结账流程编排器」串起现有要素(任务依赖+状态看板);可参考已转化的 crabwork-small-business `close-month` 骨架 |
| `journal-entry` / `journal-entry-prep` | 会计分录(借贷+凭证) | cn-fund-admin 无明确分录技能 | 中价值缺口(需本地化) | 按《企业会计准则 CAS》本地化「会计分录准备(借贷平衡+原始凭证+科目)」,落到 cn-fund-admin |
| `audit-support` / `sox-testing` | SOX 404 内控测试 | 无(SOX 为美国法) | 缺口但**强本地化** | 对标《企业内部控制基本规范》+《审计准则》,可加「内控测试/审计支持」;**绝不照搬 SOX**,需重写为中国内控框架 |
| `variance-analysis` | 差异分解(驱动+瀑布+叙述) | cn-fund-admin 有 variance-commentary;cn-equity-research 有相关 | 部分覆盖 | 低优先;可借鉴「驱动分解 + 瀑布图」增强 variance-commentary |
| `financial-statements` | 三大报表生成 + 同比 | fin-core 有 three-statement-model | 已覆盖(建模角度) | 无需引入;如需「出表」视角可在 fin-core 微扩 |
| `reconciliation` | GL vs 子账/银行/三方对账 | cn-fund-admin 有 gl-recon / break-trace | 已覆盖 | 无需引入 |

## 三、优先级汇总(供后续决策,本次不实施)

**建议优先借鉴(高价值、骨架可本地化复用):**
1. `matter-core` —— 统一法律风险分级框架(severity×likelihood + 升级阈值),横切各法律板块。
2. `cn-fund-admin` —— 月末结账流程编排器(串起现有结账要素 + 状态看板)。

**可选借鉴(中价值,需本地化):**
3. 模板化法律咨询应答(legal-response)→ cn-legal-aid / matter-core。
4. 法务每日简报 + 会议简报行动项(brief / meeting-briefing)→ matter-core。
5. 签署流程编排(signature-request,按《电子签名法》)→ cn-corporate / matter-core。
6. 会计分录准备(journal-entry,按 CAS)→ cn-fund-admin。
7. 内控测试/审计支持(SOX → 《企业内部控制基本规范》重写)→ cn-fund-admin。

**无需引入(现有已强覆盖或更贴合中国监管):**
- review-contract、triage-nda、compliance-check、vendor-check(legal);financial-statements、reconciliation(finance)。可仅借鉴其内部机制(playbook 底线库、红黄绿分流、瀑布分解),不新增技能。

## 四、注意事项

- 本文档仅为**能力缺口对照与借鉴建议**,未修改 crablaw-cn / crabfin-cn 任何文件。
- 任何并入都属独立后续工作,需:① 按对应中国法规重写(非翻译上游);② 走 skill-creator 深度优化;③ 纳入既有 matter-core / fin-core 底座的依赖与 gate 体系(如 crablaw-cn 的 Matter Gate)。
- 上游 `finance` 的 SOX、US GAAP 内容与中国准则差异大,借鉴时以**流程结构**为限,内容必须本地化。
