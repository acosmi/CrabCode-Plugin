# CrabCode 分类归并方案(纯改 marketplace.json)

- **日期**: 2026-06-21
- **依据**: `docs/audit/2026-06-20-plugin-health-and-taxonomy-audit.md` 第四节(22→8 大域)+ 第六节第 4 项(分类归并、example/examples 去重)
- **性质**: 方案存档,**尚未实施**;待拍板后再改 `marketplace.json`
- **范围**: 仅改 `.crabcode-plugin/marketplace.json` 中每个条目的 `category` 字段
- **规模**: 当前 66 个条目 / 23 个 category → 归并为 **11 个 category**,共 **40 个条目改 category,26 个不变**

---

## 零、为什么这是"可独立推进"的部分

本方案**不依赖审计第六节的两个待决项**(伞形标准、LSP/法律是否本轮合伞):

- `category` 字段**只**存在于 `marketplace.json` 的每个条目内;
- 顶层**无** `categories` 注册表;各 `plugin.json`**不带** `category` 字段;
- 因此归并 = 纯改 `marketplace.json` 的 `category` 值,**不动任何目录、插件名、其它展示字段**,零结构风险。

> LSP 合伞(审计第 3 项)会改目录结构 + 条目数,受待决项约束,**不在本方案内**。本方案下 12 个 LSP 仍是 12 个独立条目,只是 category 统一为 `language-server`(本就如此,无需改)。

## 五、设计取舍(已拍定,可推翻)

1. **example/examples 去重**:两个插件(crabcode-example-plugin = 演示骨架;crabcode-example-skills = 12 个 skill 样例集)是**不同插件**,非同一插件重复。两者都归 `essentials`,单复数分类自然消失,**不删任何插件**。
2. **skill-creator 归 `dev-tools`**:与 plugin-dev / mcp-server-dev / agent-sdk-dev 同属"造扩展",语义比塞进杂项更准。
3. **session-style 保留**:有 2 个成员(非单例)、语义清晰(会话呈现风格),保留比硬并干净、改动更小。
4. **essentials / showcase 拆分**:`essentials` = 元/基础(setup / memory / 示例);`showcase` = 能力展示(playground / math-olympiad / cwc-makers)。比把它们混成一个长尾域语义更纯。

## 六、不在本方案内(留待后续决策)

- **LSP 合伞**(审计第 3 项):改目录结构,受待决项约束。
- **crablaw-cn 伞形重构 + 元数据补全**(审计第 2、5 项):审计已定"暂缓"。
- **finance / 法律 是否进一步合成"行业工作流"大域**:本方案让 `finance`、`legal-workflow` 各自独立(各 ≥4,非单例,语义清晰),暂不合并。
- **伞形标准拍板**(待决项 1)、**LSP/法律本轮是否合伞**(待决项 2)。
