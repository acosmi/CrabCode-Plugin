## 测试夹具与 QA 调度（本轮发布阻断修复）

**根因**：业务夹具无条件调用完整 Chromium/Nu QA + Bun 多测并发/60s 预算 与 单槽 Playwright 结构性冲突，导致 IPC `ENOENT` 与超时。

**修复**：

- `createReviewedContent` 默认 `render-only`；需要门禁的用例显式 `deliveryMode: 'verified'`
- `MEDIAOPS_QA_MODE=full|static|off`：生产默认 full；业务测 static；负例 off
- 进程内共享 Chromium、跨进程文件锁、launch/goto/段超时、IPC 归类为 `qa_infrastructure_failed`
- 脚本拆分：`test` / `test:qa` / `qa:release` / `test:all`

## 门禁命令（DoD）

```bash
cd plugins/crabcode-media-ops
bun install --frozen-lockfile
bun run typecheck
bun run test
bun run test:qa
bun run build
bun run qa:release
bun run validate
```

## 微信渠道

- **自动（R-A）**：`wechat-richtext.html` 确定性生成、进 manifest、无远程脚本；本地交付/markdown 测覆盖  
- **人机（R-B，可选）**：见 `docs/releases/wechat-draft-acceptance-checklist-0.4.0.md`  
- **Known residual**：本窗口未执行真人微信后台草稿操作（浏览器安全策略拒绝 agent 控制）；不得伪造

## 残余风险

- `MEDIAOPS_QA_MODE=static` 仅用于测试加速，生产 verify 必须 full  
- 跨进程 QA 锁 stale 10 分钟；崩溃后可能短暂互斥  
- Nu/axe 非完整 WCAG 认证；真实平台清洗效果仍需人工
